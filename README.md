# Интернет-магазин «Web-Larёk»
«Web-Larёk» — это интернет-магазин с товарами для веб-разработчиков, где пользователи могут просматривать товары, добавлять их в корзину и оформлять заказы. Сайт предоставляет удобный интерфейс с модальными окнами для просмотра деталей товаров, управления корзиной и выбора способа оплаты, обеспечивая полный цикл покупки с отправкой заказов на сервер.

## Архитектура приложения

Код приложения разделен на слои согласно парадигме MVP (Model-View-Presenter), которая обеспечивает четкое разделение ответственности между классами слоев Model и View. Каждый слой несет свой смысл и ответственность:

Model - слой данных, отвечает за хранение и изменение данных.
View - слой представления, отвечает за отображение данных на странице.
Presenter - презентер содержит основную логику приложения и  отвечает за связь представления и данных.

Взаимодействие между классами обеспечивается использованием событийно-ориентированного подхода. Модели и Представления генерируют события при изменении данных или взаимодействии пользователя с приложением, а Презентер обрабатывает эти события используя методы как Моделей, так и Представлений.

### Базовый код

#### Класс Component
Является базовым классом для всех компонентов интерфейса.
Класс является дженериком и принимает в переменной `T` тип данных, которые могут быть переданы в метод `render` для отображения.

Конструктор:
`constructor(container: HTMLElement)` - принимает ссылку на DOM элемент за отображение, которого он отвечает.

Поля класса:
`container: HTMLElement` - поле для хранения корневого DOM элемента компонента.

Методы класса:
`render(data?: Partial<T>): HTMLElement` - Главный метод класса. Он принимает данные, которые необходимо отобразить в интерфейсе, записывает эти данные в поля класса и возвращает ссылку на DOM-элемент. Предполагается, что в классах, которые будут наследоваться от `Component` будут реализованы сеттеры для полей с данными, которые будут вызываться в момент вызова `render` и записывать данные в необходимые DOM элементы.
`setImage(element: HTMLImageElement, src: string, alt?: string): void` - утилитарный метод для модификации DOM-элементов `<img>`


#### Класс Api
Содержит в себе базовую логику отправки запросов.

Конструктор:
`constructor(baseUrl: string, options: RequestInit = {})` - В конструктор передается базовый адрес сервера и опциональный объект с заголовками запросов.

Поля класса:
`baseUrl: string` - базовый адрес сервера
`options: RequestInit` - объект с заголовками, которые будут использованы для запросов.

Методы:
`get(uri: string): Promise<object>` - выполняет GET запрос на переданный в параметрах ендпоинт и возвращает промис с объектом, которым ответил сервер
`post(uri: string, data: object, method: ApiPostMethods = 'POST'): Promise<object>` - принимает объект с данными, которые будут переданы в JSON в теле запроса, и отправляет эти данные на ендпоинт переданный как параметр при вызове метода. По умолчанию выполняется `POST` запрос, но метод запроса может быть переопределен заданием третьего параметра при вызове.
`handleResponse(response: Response): Promise<object>` - защищенный метод проверяющий ответ сервера на корректность и возвращающий объект с данными полученный от сервера или отклоненный промис, в случае некорректных данных.

#### Класс EventEmitter
Брокер событий реализует паттерн "Наблюдатель", позволяющий отправлять события и подписываться на события, происходящие в системе. Класс используется для связи слоя данных и представления.

Конструктор класса не принимает параметров.

Поля класса:
`_events: Map<string | RegExp, Set<Function>>)` -  хранит коллекцию подписок на события. Ключи коллекции - названия событий или регулярное выражение, значения - коллекция функций обработчиков, которые будут вызваны при срабатывании события.

Методы класса:
`on<T extends object>(event: EventName, callback: (data: T) => void): void` - подписка на событие, принимает название события и функцию обработчик.
`emit<T extends object>(event: string, data?: T): void` - инициализация события. При вызове события в метод передается название события и объект с данными, который будет использован как аргумент для вызова обработчика.
`trigger<T extends object>(event: string, context?: Partial<T>): (data: T) => void` - возвращает функцию, при вызове которой инициализируется требуемое в параметрах событие с передачей в него данных из второго параметра.

### Данные

В приложении используются две основные сущности, которые описывают данные: товар и покупатель.

#### Интерфейс IProduct
Интерфейс описывает структуру данных товара, который можно купить в интернет-магазине.

Поля интерфейса:
- `id: string` - уникальный идентификатор товара
- `description: string` - описание товара
- `image: string` - URL изображения товара
- `title: string` - название товара
- `category: string` - категория товара
- `price: number | null` - цена товара. Может быть числом или null, если цена не указана

#### Интерфейс IBuyer
Интерфейс описывает структуру данных покупателя, необходимых для оформления заказа.

Поля интерфейса:
- `payment: TPayment` - вид оплаты. Тип `TPayment` представляет объект с полем `name`, которое может принимать значения `'card'`, `'cash'` или пустую строку `''`
- `email: string` - электронная почта покупателя
- `phone: string` - телефон покупателя
- `address: string` - адрес доставки

### Модели данных

Модели данных отвечают за хранение и работу с данными приложения. В приложении используются три модели, каждая из которых отвечает за свою зону ответственности.

#### Класс Catalog
Модель каталога товаров отвечает за хранение всех товаров, доступных для покупки в приложении, и управление товаром, выбранным для подробного отображения.

Конструктор:
Конструктор класса не принимает параметров.

Поля класса:
- `productList: IProduct[]` - массив всех товаров, доступных в каталоге. Хранит полный список товаров, полученных от сервера
- `currentProduct: IProduct | null` - товар, выбранный для подробного отображения в модальном окне. Может быть `null`, если товар не выбран

Методы класса:
- `setProducts(products: IProduct[]): void` - сохраняет массив товаров, полученный в параметрах метода, в поле `productList`
- `getProducts(): IProduct[]` - возвращает массив всех товаров из модели
- `getProductById(id: string): IProduct | undefined` - получает товар по его идентификатору, переданному в параметре метода. Возвращает объект товара или `undefined`, если товар с таким идентификатором не найден
- `setCurrentProduct(product: IProduct): void` - сохраняет товар, переданный в параметре, в поле `currentProduct` для подробного отображения
- `getCurrentProduct(): IProduct | null` - возвращает товар, выбранный для подробного отображения, или `null`, если товар не выбран

#### Класс Basket
Модель корзины отвечает за хранение товаров, выбранных покупателем для покупки, и предоставляет методы для управления содержимым корзины.

Конструктор:
Конструктор класса не принимает параметров.

Поля класса:
- `items: IProduct[]` - массив товаров, которые находятся в корзине покупателя

Методы класса:
- `getItems(): IProduct[]` - возвращает массив всех товаров, которые находятся в корзине
- `addItem(product: IProduct): void` - добавляет товар, переданный в параметре метода, в массив корзины
- `removeItem(product: IProduct): void` - удаляет товар, переданный в параметре метода, из массива корзины
- `clear(): void` - очищает корзину, удаляя все товары из массива
- `getTotal(): number` - возвращает общую стоимость всех товаров в корзине. Суммирует цены всех товаров, учитывая, что цена может быть `null`
- `getCount(): number` - возвращает количество товаров в корзине
- `contains(id: string): boolean` - проверяет наличие товара в корзине по его идентификатору, переданному в параметре метода. Возвращает `true`, если товар с таким идентификатором находится в корзине, и `false` в противном случае

#### Класс Buyer
Модель покупателя отвечает за хранение данных покупателя, необходимых для оформления заказа, и валидацию этих данных.

Конструктор:
Конструктор класса не принимает параметров.

Поля класса:
- `payment: TPayment` - вид оплаты, выбранный покупателем
- `address: string` - адрес доставки
- `phone: string` - телефон покупателя
- `email: string` - электронная почта покупателя

Методы класса:
- `setData(data: Partial<IBuyer>): void` - сохраняет данные покупателя в модели. Принимает объект с частичными данными покупателя, что позволяет сохранить только одно или несколько полей, не удаляя при этом значения других полей, которые уже могут храниться в классе
- `getData(): IBuyer` - возвращает объект со всеми данными покупателя
- `clear(): void` - очищает все данные покупателя, сбрасывая значения всех полей
- `validate(): Partial<Record<keyof IBuyer, string>>` - выполняет валидацию данных покупателя. Возвращает объект, в котором ключи соответствуют полям класса, а значения - тексты ошибок валидации. Если поле не содержит ошибок, соответствующее свойство в объекте отсутствует. Поле считается валидным, если оно не пустое. Для поля `payment` проверяется, что выбран вид оплаты (значение не пустое)
### Слой коммуникации

Слой коммуникации отвечает за взаимодействие приложения с внешними сервисами, в частности с API сервера. Класс этого слоя использует композицию для работы с базовым классом `Api`, который предоставляет методы для выполнения HTTP-запросов.

#### Интерфейс IProductsResponse
Интерфейс описывает структуру ответа сервера при запросе каталога товаров.

Поля интерфейса:
- `total: number` - общее количество товаров на сервере
- `items: IProduct[]` - массив товаров, полученных с сервера

#### Интерфейс IOrderRequest
Интерфейс описывает структуру данных, отправляемых на сервер при оформлении заказа.

Поля интерфейса:
- `payment: TPayment` - вид оплаты
- `email: string` - электронная почта покупателя
- `phone: string` - телефон покупателя
- `address: string` - адрес доставки
- `items: string[]` - массив идентификаторов товаров в заказе
- `total: number` - общая стоимость заказа

#### Класс ShopApi
Класс представляет слой коммуникации с сервером и отвечает за получение данных о товарах и отправку заказов на сервер. Класс использует композицию, принимая в конструкторе объект, реализующий интерфейс `IApi`, и использует его методы для выполнения HTTP-запросов.

Конструктор:
`constructor(api: IApi)` - принимает объект, соответствующий интерфейсу `IApi`, который будет использоваться для выполнения запросов к серверу.

Поля класса:
- `api: IApi` - объект для выполнения HTTP-запросов к серверу

Методы класса:
- `getProducts(): Promise<IProduct[]>` - выполняет GET запрос на эндпоинт `/product/` и возвращает промис с массивом товаров. Метод использует метод `get` объекта `api` для получения данных с сервера, извлекает массив товаров из ответа типа `IProductsResponse` и возвращает его
- `postOrder(order: IOrderRequest): Promise<IOrderResponse>` - выполняет POST запрос на эндпоинт `/order/` и передаёт в него данные заказа, полученные в параметре метода. Метод использует метод `post` объекта `api` для отправки данных на сервер и возвращает промис с ответом сервера типа `IOrderResponse`, содержащим идентификатор заказа и общую стоимость

### Слой Представления (View)

Слой представления отвечает за отображение данных на странице и взаимодействие с пользователем. Все компоненты представления наследуются от базового класса `Component` и используют событийную модель для уведомления о действиях пользователя.

#### Базовый класс Card
Абстрактный базовый класс для всех карточек товара. Содержит общий функционал для работы с карточками товаров.

Конструктор:
`constructor(container: HTMLElement, events: IEvents)` - принимает контейнер для карточки (клонированный из шаблона в презентере) и обязательный объект для работы с событиями. Инициализирует обязательные поля `_title` и `_price` с помощью `ensureElement`.

Поля класса:
- `container: HTMLElement` - корневой DOM-элемент карточки (наследуется от Component)
- `_title: HTMLElement` - обязательный элемент с названием товара
- `_price: HTMLElement` - обязательный элемент с ценой товара
- `events: IEvents` - объект для генерации событий (обязательный)

Методы класса:
- `render(data?: Partial<IProduct>): HTMLElement` - абстрактный метод, должен быть реализован в дочерних классах. Возвращает корневой элемент карточки
- `setTitle(title: string): void` - устанавливает название товара в элемент `_title`
- `setPrice(price: number | null): void` - устанавливает цену товара. Если цена `null`, отображает "Бесценно", иначе отображает цену в формате "{цена} синапсов"

#### Базовый класс CardWithImage
Абстрактный промежуточный класс для карточек с изображением и категорией. Наследуется от `Card`.

Конструктор:
`constructor(container: HTMLElement, events: IEvents)` - принимает контейнер и объект для работы с событиями. Инициализирует поля `_category` и `_image` с помощью `ensureElement`.

Поля класса:
- `_category: HTMLElement` - элемент с категорией товара
- `_image: HTMLImageElement` - элемент с изображением товара

Методы класса:
- `setCategory(category: string): void` - устанавливает категорию товара и применяет соответствующий модификатор для фона категории из `categoryMap`
- `setImage(src: string, alt?: string): void` - устанавливает изображение товара. Добавляет к пути изображения базовый URL из `CDN_URL` и вызывает метод `setImage` базового класса Component

#### Класс CardCatalog
Компонент карточки товара для отображения в каталоге на главной странице. Наследуется от `CardWithImage`.

Конструктор:
`constructor(container: HTMLElement, events: IEvents)` - принимает контейнер (клонированный из шаблона `#card-catalog` в презентере) и обязательный объект для работы с событиями. Устанавливает обработчик клика на карточку.

Поля класса:
- Все поля наследуются от `Card` и `CardWithImage`
- `_clickHandler?: () => void` - приватный колбэк для обработки клика

Методы класса:
- `render(data?: Partial<IProduct>): HTMLElement` - отображает карточку товара в каталоге с данными из параметра. Создает колбэк с данными товара через замыкание. При клике на карточку генерирует событие `'card:select'` с данными товара из колбэка

#### Класс CardPreview
Компонент карточки товара для детального просмотра в модальном окне. Наследуется от `CardWithImage`.

Конструктор:
`constructor(container: HTMLElement, events: IEvents)` - принимает контейнер (клонированный из шаблона `#card-preview` в презентере) и обязательный объект для работы с событиями. Инициализирует поля `_text` и `_button` с помощью `ensureElement`. Устанавливает обработчик клика на кнопку.

Поля класса:
- Все поля наследуются от `Card` и `CardWithImage`
- `_text: HTMLElement` - элемент с описанием товара
- `_button: HTMLButtonElement` - кнопка "Купить" или "Удалить из корзины"
- `_buttonClickHandler?: () => void` - приватный колбэк для обработки клика на кнопку

Методы класса:
- `render(data?: Partial<IProduct & { inBasket: boolean }>): HTMLElement` - отображает детальную информацию о товаре. Создает колбэк с данными товара и состоянием корзины через замыкание. Если товар в корзине (`inBasket === true`), кнопка показывает "Удалить из корзины", иначе "В корзину". Если цена `null`, кнопка заблокирована с текстом "Недоступно". При клике на кнопку генерирует событие `'card:add'` или `'card:remove'` в зависимости от состояния корзины из колбэка

#### Класс CardBasket
Компонент карточки товара для отображения в корзине. Наследуется от `Card`.

Конструктор:
`constructor(container: HTMLElement, events: IEvents)` - принимает контейнер (клонированный из шаблона `#card-basket` в презентере) и обязательный объект для работы с событиями. Инициализирует поля `_index` и `_button` с помощью `ensureElement`. Устанавливает обработчик клика на кнопку удаления.

Поля класса:
- Все поля наследуются от `Card`
- `_index: HTMLElement` - элемент с номером товара
- `_button: HTMLButtonElement` - кнопка удаления товара
- `_buttonClickHandler?: () => void` - приватный колбэк для обработки клика на кнопку

Методы класса:
- `render(data?: Partial<IProduct & { index: number }>): HTMLElement` - отображает товар в корзине с номером позиции. Создает колбэк с данными товара через замыкание. Устанавливает номер позиции в элемент `_index`. При клике на кнопку удаления генерирует событие `'card:remove'` с данными товара из колбэка

#### Базовый класс Form
Абстрактный базовый класс для всех форм. Содержит общий функционал для работы с формами.

Конструктор:
`constructor(container: HTMLElement, events: IEvents)` - принимает контейнер (клонированный из шаблона в презентере) и обязательный объект для работы с событиями. Инициализирует поля `_submitButton` и `_errors` с помощью `ensureElement`.

Поля класса:
- `_submitButton: HTMLButtonElement` - кнопка отправки формы
- `_errors: HTMLElement` - элемент для отображения ошибок
- `events: IEvents` - объект для генерации событий (обязательный)

Методы класса:

#### Класс Modal
Компонент модального окна. Отвечает за отображение контента в модальном окне и его открытие/закрытие. Наследуется от `Component<void>`.

Конструктор:
`constructor(container: HTMLElement)` - принимает элемент с классом `modal` и id `modal-container`. Инициализирует поля `_content` и `_closeButton` с помощью `ensureElement`. Устанавливает обработчики событий для закрытия модального окна: по клику на кнопку закрытия и по клику вне модального окна. Предотвращает скролл модального окна.

Поля класса:
- `container: HTMLElement` - корневой элемент модального окна (наследуется от Component)
- `_content: HTMLElement` - контейнер для контента модального окна
- `_closeButton: HTMLButtonElement` - кнопка закрытия модального окна

Методы класса:
- `render(): HTMLElement` - возвращает корневой элемент модального окна
- `setContent(content: HTMLElement): void` - устанавливает контент в модальное окно, заменяя все дочерние элементы контейнера `_content` на переданный элемент
- `open(): void` - открывает модальное окно, добавляя модификатор `modal_active` к элементу `container`. Блокирует скролл страницы, устанавливая `document.body.style.overflow = 'hidden'`

- `containsBasket(): boolean` - проверяет, содержит ли модальное окно компонент корзины. Возвращает `true`, если в `_content` найден элемент с классом `basket`

### События приложения

События генерируются компонентами представления и моделями данных для уведомления о действиях пользователя и изменениях данных.

#### События от моделей данных:
- `'catalog:changed'` - генерируется при изменении каталога товаров (массив товаров обновлен)
- `'catalog:product-changed'` - генерируется при изменении выбранного для просмотра товара
- `'basket:changed'` - генерируется при изменении содержимого корзины (добавление, удаление, очистка)
- `'buyer:changed'` - генерируется при изменении данных покупателя

#### События от компонентов представления:
- `'card:select'` - генерируется при клике на карточку товара в каталоге. Данные: `{ product: IProduct }`
- `'card:add'` - генерируется при нажатии кнопки "Купить" в карточке товара. Данные: `{ product: IProduct }`
- `'card:remove'` - генерируется при нажатии кнопки удаления товара из корзины. Данные: `{ product: IProduct }`
- `'header:basket'` - генерируется при клике на кнопку корзины в шапке. Данные отсутствуют
- `'basket:order'` - генерируется при нажатии кнопки "Оформить" в корзине. Данные отсутствуют
- `'order:change'` - генерируется при изменении полей формы оформления заказа. Данные: `Partial<IBuyer>` (только измененное поле)
- `'order:submit'` - генерируется при отправке формы выбора оплаты и адреса. Данные отсутствуют
- `'contacts:change'` - генерируется при изменении полей формы контактов. Данные: `Partial<IBuyer>` (только измененное поле)
- `'contacts:submit'` - генерируется при отправке формы контактов. Данные отсутствуют
- `'success:close'` - генерируется при нажатии кнопки закрытия сообщения об успехе. Данные отсутствуют

### Презентер

Презентер содержит основную логику приложения и отвечает за связь между слоями данных и представления. В данном проекте презентер реализован в основном файле приложения `main.ts` в виде обработчиков событий.

Презентер подписывается на события от моделей данных и компонентов представления, обрабатывает их и вызывает соответствующие методы моделей или представлений для обновления состояния приложения.

Основные функции презентера:
- Инициализация приложения: создание экземпляров моделей, компонентов представления и API
- Обработка событий от моделей данных: обновление представления при изменении данных
- Обработка событий от компонентов представления: выполнение действий с данными и обновление представления
- Управление модальными окнами: открытие и закрытие модальных окон в зависимости от действий пользователя
- Валидация данных: проверка данных форм перед отправкой заказа
- Отправка заказа на сервер: формирование данных заказа и отправка через API

Презентер не генерирует события, а только обрабатывает их, обеспечивая связь между слоями приложения согласно паттерну MVP.
